# B.1 — Config + ToolAdapter + ClaudeAdapter + registry + CLI `--tools=`

**Bead:** `agents-config-w1qls.2.1`

## Context

B.1 is the gate between A.\* (model + IOPort, already shipped) and the rest of Epic B (sync engine, template handling) plus much of D/F/G. It introduces the tool-adapter contract that every tool-specific behavior in the installer (source location, destination, detection, namespace iteration, post-staging transforms) hangs off of.

Predecessors landed:

- **A.2** (`core/model.py`): `Tool` StrEnum, `Provenance`, `FileKind`, `StagedItem`, `StagingPlan`, `IncludeDirective`, `Counters`.
- **A.3** (`core/io_port.py`): `IOPort` Protocol + `TerminalIO` + `ScriptedIO` test fake.

B.1 ships only `ToolAdapter` + `ClaudeAdapter` + the registry + the minimal `Config` dataclass + CLI `--tools=` parsing. Sync (B.2), staging (C.1), other adapters (D.\*), plugins (F.\*), TOML config (G.2), and `--dump-stage` (G.6) all build on this gate.

## Decisions

### Architecture

- **Protocol + concrete classes** for adapters — direct mirror of the IOPort/TerminalIO/ScriptedIO precedent. `tools/base.py` defines a `@runtime_checkable` Protocol; `tools/claude.py` defines `ClaudeAdapter` as a stateless class with methods; `tools/registry.py` exposes a module-level `dict[Tool, ToolAdapter]` plus lookup helpers. Considered: module-as-adapter (flatter but loses Protocol's mypy support and uniform fake-test shape). Rejected — uniformity with IOPort idiom is worth the small ergonomic cost.
- **Registry-as-truth** for "valid tool." `--tools=opencode` fails in B.1 with `Unknown tool: 'opencode' (valid: claude)`. As each adapter lands in D.\*, the registry grows and that tool becomes valid automatically. Fail-fast UX; no late mystery error.
- **Module-level registry dict** keyed by `Tool` enum. Plugins use the parallel string-keyed pattern in F.1; tools and plugins do not share infra.

### Deliberate divergences from install.sh

1. **`is_detected` probes `~/.claude/settings.json`**, not the directory. The marker file is Claude Code's actual initialization signal; a directory could exist for incidental reasons (misfired `mkdir`, accidental inspection). Stronger probe → fewer false positives.
2. **Claude is no longer always auto-detected.** Bash special-cased `TOOLS=(claude); for tool in codex gemini; do [[ -d ~/.$tool ]] && TOOLS+=...; done` — claude was unconditional, others probed. Python is uniform: every adapter, including claude, probes its dest marker. If a user runs the installer on a host without Claude Code initialized, `--tools=claude` is the override.
3. **Empty `--tools=` is a usage error.** Bash silently produced an empty tool list and ran a no-op install. Python raises `ValueError` → CLI exits 2 with `installer: --tools= requires at least one tool`.
4. **CSV whitespace is stripped, duplicates are deduped (first-occurrence wins).** User-facing improvements; neither changes observable install output.
5. **`Config` is frozen and minimal.** B.1 ships only `home` and `tools`. Subsequent stories add fields as their behavior requires (`repo_root` in B.2, `dry_run` / `verbose` in B.2 onward).

These divergences are documented for retirement of golden-master parity in H.5 — they collectively form the first deliberate parity-break batch. Smoke-test impact: zero on Scott's machine (Claude Code initialized); the golden-master "bare install" scenario accepts the divergence rather than fixturing around it (decision: this design's brainstorm session log).

### YAGNI deferrals

- `Config.repo_root` field — B.2 (sync engine) is the first consumer; field added there.
- `IOPort` wiring in `cli.py` — B.2 introduces it when there is something to say beyond an error string.
- Plugin auto-detection (`--plugins=` parsing, plugin registry) — F.1.
- `installer.toml` loading — G.2.

## `tools/base.py` — ToolAdapter Protocol

```python
from __future__ import annotations
from pathlib import Path
from typing import TYPE_CHECKING, Protocol, runtime_checkable

if TYPE_CHECKING:
    from installer.core.io_port import IOPort
    from installer.core.model import StagingPlan


@runtime_checkable
class ToolAdapter(Protocol):
    """Tool-specific behaviour the engine consults. One concrete adapter
    per `Tool` enum value; registry-wired."""

    name: str
    detection_signal: str

    def source_dir(self, repo_root: Path) -> Path: ...
    def dest_dir(self, home: Path) -> Path: ...
    def is_detected(self, home: Path) -> bool: ...
    def scoped_namespaces(self) -> tuple[str, ...]: ...
    def should_install_namespace(self, namespace: str, source: str) -> bool: ...
    def post_staging_transforms(self, plan: StagingPlan, io: IOPort) -> StagingPlan: ...
```

`IOPort` and `StagingPlan` are imported under `TYPE_CHECKING` so the Protocol module stays import-cheap on CLI paths that never reach the staging engine.

Six methods on the Protocol from day one. Adding methods to a Protocol later is a breaking change for every implementer; better to commit now. Each method's first real consumer is named in the "Forward pragma-removal" section.

## `tools/claude.py` — ClaudeAdapter

```python
from __future__ import annotations
from pathlib import Path
from typing import TYPE_CHECKING

if TYPE_CHECKING:
    from installer.core.io_port import IOPort
    from installer.core.model import StagingPlan


class ClaudeAdapter:
    """Adapter for Anthropic's Claude Code. Probes ~/.claude/settings.json
    as the detection marker — deliberate divergence from install.sh's
    'always include claude' rule (see §Decisions)."""

    name: str = "claude"
    detection_signal: str = "~/.claude/settings.json"

    def source_dir(self, repo_root: Path) -> Path:  # pragma: no cover  # exercised by w1qls.2.2 (B.2)
        return repo_root / "src" / "user" / ".claude"

    def dest_dir(self, home: Path) -> Path:  # pragma: no cover  # exercised by w1qls.2.2 (B.2)
        return home / ".claude"

    def is_detected(self, home: Path) -> bool:
        return (home / ".claude" / "settings.json").is_file()

    def scoped_namespaces(self) -> tuple[str, ...]:  # pragma: no cover  # exercised by w1qls.3.1 (C.1)
        return ("commands", "skills", "agents", "rules")

    def should_install_namespace(  # noqa: ARG002  # protocol parameter; uniform across adapters
        self, namespace: str, source: str
    ) -> bool:  # pragma: no cover  # exercised by w1qls.3.1 (C.1)
        return True

    def post_staging_transforms(  # noqa: ARG002  # protocol parameter; uniform across adapters
        self, plan: StagingPlan, io: IOPort
    ) -> StagingPlan:  # pragma: no cover  # exercised by w1qls.4.4 (D.4)
        return plan
```

`detection_signal` carries the human-readable probe path for diagnostics/help text; `is_detected` performs the actual filesystem check. `IOPort` and `StagingPlan` are imported under `TYPE_CHECKING` so this module does not pull the staging-engine surface onto cheap CLI paths.

## `tools/registry.py`

```python
from __future__ import annotations

from installer.core.model import Tool
from installer.tools.base import ToolAdapter
from installer.tools.claude import ClaudeAdapter

_REGISTRY: dict[Tool, ToolAdapter] = {
    Tool.CLAUDE: ClaudeAdapter(),
}


class UnknownToolError(ValueError):
    """Raised when a CLI-supplied tool name is not in the registry.
    Structured attrs (.name, .valid) so callers and tests assert on
    data, not on the message string."""

    def __init__(self, name: str, valid: tuple[str, ...]) -> None:
        super().__init__(f"Unknown tool: {name!r} (valid: {', '.join(valid)})")
        self.name = name
        self.valid = valid


def known_tools() -> tuple[Tool, ...]:
    """Sorted tuple of registered Tool values. Drives auto-detect iteration
    and `--tools=` validation error messages."""
    return tuple(sorted(_REGISTRY.keys(), key=lambda t: t.value))


def get_adapter(tool: Tool) -> ToolAdapter:
    """Lookup. Raises KeyError on miss — callers expected to have
    validated via parse_tool_name first."""
    return _REGISTRY[tool]


def parse_tool_name(name: str) -> Tool:
    """Translate a `--tools=` CSV element to its Tool enum value. Filters
    on registry membership, not just enum membership (registry-is-truth)."""
    valid_names = tuple(t.value for t in known_tools())
    try:
        candidate = Tool(name)
    except ValueError:
        raise UnknownToolError(name, valid_names) from None
    if candidate not in _REGISTRY:
        raise UnknownToolError(name, valid_names)
    return candidate
```

## `config.py`

```python
from __future__ import annotations
from dataclasses import dataclass
from pathlib import Path

from installer.core.model import Tool
from installer.tools.registry import get_adapter, known_tools, parse_tool_name


@dataclass(frozen=True, slots=True)
class Config:
    """Resolved installer configuration. Frozen so the engine can pass it
    freely without defensive copies. B.1 scope: only fields needed for
    tool selection. Later stories add fields as their behaviour requires."""

    home: Path
    tools: tuple[Tool, ...]


def resolve_tools(*, home: Path, override_csv: str | None) -> tuple[Tool, ...]:
    """Translate the `--tools=` CLI value into the resolved tool tuple.

    - `override_csv is None` → auto-detect: walk known_tools(), keep
      those whose adapter reports `is_detected(home) is True`.
    - `override_csv == ""` (or whitespace-only) → ValueError.
    - Otherwise → split on commas, strip whitespace, validate each via
      `parse_tool_name`, dedupe preserving first occurrence, preserve
      user-supplied order.
    """
    if override_csv is None:
        return tuple(t for t in known_tools() if get_adapter(t).is_detected(home))

    if override_csv.strip() == "":
        raise ValueError("--tools= requires at least one tool")

    seen: dict[Tool, None] = {}
    for raw in override_csv.split(","):
        tool = parse_tool_name(raw.strip())
        seen.setdefault(tool, None)
    return tuple(seen.keys())
```

## `cli.py` — updated

```python
from __future__ import annotations
import argparse
import sys
from pathlib import Path

from installer.config import Config, resolve_tools
from installer.tools.registry import UnknownToolError, known_tools


def _build_parser() -> argparse.ArgumentParser:
    parser = argparse.ArgumentParser(
        prog="installer",
        description="Install agent configurations for AI coding assistants.",
    )
    parser.add_argument(
        "--tools",
        metavar="CSV",
        default=None,
        help=(
            "Comma-separated tool list to install "
            f"(valid: {', '.join(t.value for t in known_tools())}). "
            "Default: auto-detect against $HOME."
        ),
    )
    return parser


def main(argv: list[str] | None = None, *, home: Path | None = None) -> int:
    args = _build_parser().parse_args(argv)
    resolved_home = home if home is not None else Path.home()

    try:
        tools = resolve_tools(home=resolved_home, override_csv=args.tools)
    except (UnknownToolError, ValueError) as exc:
        sys.stderr.write(f"installer: {exc}\n")
        return 2

    Config(home=resolved_home, tools=tools)
    return 0
```

`main` accepts `home` keyword for hermetic test injection — matches the IOPort `Console` injection precedent. Production callers pass nothing; tests pass `tmp_path`.
